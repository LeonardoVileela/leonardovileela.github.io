Ch'aska API

Based on simple REST principles, the Ch'aska API endpoints return JSON metadata about Ch'aska Store.

The base address of Web API is http://localhost:8080.

Requests

The Ch'aska API is based on REST principles. Data resources are accessed via standard HTTPS requests in UTF-8 format to an API endpoint. Where possible, Web API uses appropriate HTTP verbs for each action:

Method Action
GET Retrieves resources
POST Creates resources
PUT Changes and/or replaces resources or collections
DELETE Deletes resources

Responses

Web API returns all response data as a JSON object.

Response Status Codes

Web API uses the following response status codes, as defined in the

Status Code Description
200 OK - The request has succeeded. The client can read the result of the request in the body and the headers of the response.
201 Created - The request has been fulfilled and resulted in a new resource being created.
202 Accepted - The request has been accepted for processing, but the processing has not been completed.
204 No Content - The request has succeeded but returns no message body.
304 Not Modified.
400 Bad Request - The request could not be understood by the server due to malformed syntax. The message body will contain more information.
401 Unauthorized - The request requires user authentication or, if the request included authorization credentials, authorization has been refused for those credentials.
403 Forbidden - The server understood the request, but is refusing to fulfill it.
404 Not Found - The requested resource could not be found. This error can be due to a temporary or permanent condition.
500 Internal Server Error.
502 Bad Gateway - The server was acting as a gateway or proxy and received an invalid response from the upstream server.
503 Service Unavailable - The server is currently unable to handle the request due to a temporary condition which will be alleviated after some delay. You can choose to resend the request again.

Documentation to use API

Here is a documentation for use and examples.

Authentication

URL: http://localhost:8080/login

METHOD: POST

Key Value Type Value Description
username string Username for authentication
password string Password for authentication

Here is an Example.

{
    "username": "admin",
    "password": "123"
}

Client GET

URL: http://localhost:8080/api/clients

METHOD: GET

Key Value Type Value Description

Sales quantity statistics GET

URL: http://localhost:8080/api/salesCont/{id}

METHOD: GET

Key Value Type Value Description
salesCont int Sales quantity of user in current month
percentageTotal int Percentage compared to last month
percentage boolean 'percentage' true is positive percentage, 'percentage' false is negative percentage

    "percentageTotal": 100,
    "percentage": true
}
